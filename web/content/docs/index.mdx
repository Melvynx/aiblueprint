AIBlueprint CLI manages local AI coding configuration. It installs a shared `.agents` folder, renders Codex agent files, links tool folders, handles premium configuration updates, and keeps recoverable backups around risky changes.

```bash
npx aiblueprint-cli@latest agents setup
```

## Command map

| Command | Purpose |
| --- | --- |
| `aiblueprint agents` | Main AI coding configuration command group. |
| `aiblueprint ai-coding` | Legacy alias for `agents`. |
| `aiblueprint claude-code` | Legacy alias for `agents`. |
| `aiblueprint configs` | Named saves, restores, undo, and backup inspection. |
| `aiblueprint openclaw` | OpenClaw configuration command group. |

## Folder options

Most configuration commands accept these shared options:

| Option | Meaning |
| --- | --- |
| `-f, --folder <path>` | Root folder containing `.claude`, `.codex`, and `.agents`. Defaults to `$HOME`. |
| `--claudeCodeFolder <path>` | Override the Claude Code folder. |
| `--codexFolder <path>` | Override the Codex folder. |
| `--agentsFolder <path>` | Override the shared agents folder. |

## Recommended first run

Use `agents setup` for a complete install. Use `configs save` before experimenting with local configuration changes.

```bash
npx aiblueprint-cli@latest agents setup
npx aiblueprint-cli@latest configs save before-experiment
```
